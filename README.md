## Predictive Maintenance of Aircraft Landing Gear Using Physics-Informed Digital Twin Data and Machine Learning
---
## Abstract
This capstone project investigates predictive maintenance of aircraft landing gear using physics-informed digital twin data and machine learning techniques. The study builds on a high-fidelity digital twin framework that generated simulated landing events under multiple fault conditions. Tree-based ensemble learning methods are employed due to their suitability for structured, event-level data. Random Forest and XGBoost models are applied for multi-class fault diagnosis, while regression variants of the same models are used for remaining useful life (RUL) prediction. The results demonstrate that physics-informed features combined with ensemble learning models can effectively support fault identification and RUL estimation, highlighting the feasibility of condition-based maintenance strategies for aircraft landing gear systems and other safety-critical components.

**Keywords:** machine learning, predictive maintenance, digital twin, fault diagnosis, remaining useful life

---
## Problem Statement
Landing gear failures often develop gradually and may not be detected by traditional time-based maintenance approaches. The research question addressed in this project is:

> Can machine learning models trained on physics-informed digital twin data reliably detect abnormal landing gear conditions and predict remaining useful life at the event level?

## Data Sources

### Primary Dataset
Sani, M. B. (2026). *Aircraft Landing Gear Digital Twin Dataset (PDM/SHM)* [Data set]. Kaggle.  
https://www.kaggle.com/datasets/mohammedbellosani/aircraft-landing-gear-digital-twin-datasetpdmshm

- 1,500-record balanced subset used for primary model training and evaluation
- 9 features per landing event: `RunID`, `Max_Deflection`, `Max_Velocity`, `Settling_Time`, `Mass`, `K_Stiffness`, `B_Damping`, `Fault_Code`, `RUL`
- 4 fault classes (375 records each): Normal (0), Nitrogen Leak (1), Worn Seal (2), Early Structural Degradation (3)
- Generated by a MATLAB Simscape Multibody digital twin of the Dornier 228 aircraft landing gear

### Secondary Dataset (Validation / Generalization)
Sani, M. B. (2026). *Dornier 228 Synthetic Aircraft LandingGear Dataset* [Data set]. Kaggle.  
https://www.kaggle.com/datasets/mohammedbellosani/dornier-228-synthetic-aircraft-landinggear-dataset

- 9,500 physics-based landing cycles generated for LSTM predictive maintenance (CS-23.473 compliant)
- 23 features per landing event including: `SinkRate_ms`, `Friction_Coef`, `Temp_C`, `Health_Factor`, `Max_Accel_g`, `Fatigue_Damage`, `Maintenance_Cost_USD`, and others
- Used to assess generalizability of models trained on the primary dataset

Both datasets are published by the same author, sourced from the same Dornier 228 digital twin framework, and are used in accordance with their published licenses. Neither dataset is redistributed in this repository.

## Methodology

**Data Representation**
- Each row corresponds to one independent landing event (event-level, tabular structure)
- No temporal continuity assumed between events

**Feature Engineering**
- Physics-informed summary features describing landing dynamics and degradation
- Derived quantities including natural frequency (ωₙ) and damping ratio (ζ) computed from structural parameters (K_Stiffness, B_Damping, Mass) during exploratory analysis

**Models**
- Random Forest Classifier / Regressor
- XGBoost Classifier / Regressor
- Logistic Regression (baseline, classification only)

**Training Strategy**
- Hold-out train/test split at event level
- Cross-validation for stability assessment
- Default or minimally adjusted hyperparameters (comparison reflects structural algorithm suitability, not tuning)

**Evaluation Metrics**
- Classification: Accuracy, Precision, Recall, F1-Score
- Regression: Mean Absolute Error (MAE), Root Mean Squared Error (RMSE)
- Feature importance analysis for model interpretability

**Tech Stack**
- Python (pandas, NumPy, scikit-learn, xgboost)
- Offline batch processing on local workstation
